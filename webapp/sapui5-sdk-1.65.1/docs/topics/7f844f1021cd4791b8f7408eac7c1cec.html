Setting the Table Type

In the manifest.json file, and through annotations, you can control which table type is rendered in the list report and on the object page.

The following tableType settings are available:
  • ResponsiveTable
  • GridTable
  • AnalyticalTable
  • TreeTable
Note

  • Grid tables, analytical tables, and tree tables cannot be rendered on smart phones.

  • If you use tableType together with the table-specific flags that were used up to SAP UI5 1.48 (for example, gridTable: true, treeTable: true), tableType has a higher priority, and the old flags are ignored.

  • List report only: If you use tableType: AnalyticalTable, set the annotation sap:semantics to aggregate for the specified entity type. Note that sap:semantics is a back-end entity type definition and cannot be changed in the SAP Web IDE.

  • Object page only: If you do not maintain a tableType flag, and if sap:semantics has been set in the back-end, an analytical table is rendered.

Examples

Set tableType to the required value in the sap.ui.generic.app section of the manifest.json:

Example for the list report:

"sap.ui.generic.app": {
			"pages": [{
				"entitySet": "Zfarvd_Bs_Hd_Bo",
				"component": {
					"name": "sap.suite.ui.generic.template.ListReport",
					"list": true,
					"settings": {
						"tableType": "GridTable",
						"hideTableVariantManagement": false
					}
				},

Examples for the object page:

"sap.ui.generic.app": {
        "pages": [
            {
                "entitySet": "STTA_C_MP_Product",
                "component": {
                    "name": "sap.suite.ui.generic.template.ListReport",
                    "list": true
                },
                "pages": [
                    {
                        "entitySet": "STTA_C_MP_Product",
                        "component": {
                            "name": "sap.suite.ui.generic.template.ObjectPage"
                            "settings": {
                                "sections": {
                                    "to_ProductText::com.sap.vocabularies.UI.v1.LineItem": {
                                        "navigationProperty": "to_ProductText",
                                        "entitySet": "STTA_C_MP_ProductText",
                                        "tableType": "TreeTable"
                                    }
                                }
                            }
                        },
tableType can also be maintained within settings.
Note If you maintain the tableType flag inside sections, this has a higher priority than the tableType flag within settings.

                              "_version": "1.3.0",
