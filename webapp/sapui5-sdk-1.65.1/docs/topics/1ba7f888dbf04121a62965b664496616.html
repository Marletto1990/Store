Placing Reuse Component Instances on the Object Page

Each reuse component instance defined for an object page of an SAP Fiori elements-based app is realized by a section that contains a subsection which in turn contains the content of the reuse component.

By default, these sections are added to the end of the object page in the same order as the reuse component instances are defined in the manifest. You can move these sections using the SAP Visual Editor.

Adding a Reuse Component as a Sub-Section to an Existing Section

You can add a reuse component as a last subsection to any existing section, using the manifest property "leadingSectionIdOrPath" to link it with the desired section. In the annotations, collection facets are used with their IDs, and reference facets are used with their annotation paths. Therefore, you need to provide either the ID or the path of the required facet as a value to this property. During templating, the reuse component is attached as a subsection to the section. See an example manifest below:

Sample Code 
"embeddedComponents": {
                    "tableTest": {
                        "id":"tableTest",
                        "componentName": "STTA_MP.reuseComponents.tableTest",
                        "title": "{{SalesPriceReuse}}",
                        "leadingSectionIdOrPath": "GeneralInformation",
                        "settings": {
                            "navigationProperty": "to_ProductSalesPrice"
                        }
                    }
                }

Grouping Reuse Components into a Single Section

You can group reuse components into a single section. Each reuse component has an ID. You can choose one of them to be the leading component, and the others can be grouped with it. See the example manifest below.

Use the "groupTitle" property for the title of the grouped reuse components. If there is no title, the existing title is used as the title of the group (section). If the configuration is incorrect, such as using a reuse component in a grouping which has already been grouped with another one, the fallback solution is to show it as a separate section.

Sample Code 
"embeddedComponents": {
                    "stateTest": {
                        "id": "stateTest",
                        "componentUsage": "stateTest",
                        "title": "{{stateReuse}}",
                        "groupTitle":"{{reuseGroupTitle}}"
                    },
                    "situationsTest": {
                        "id": "situationsTest",
                        "componentName": "STTA_MP.reuseComponents.situationsTest",
                        "title": "{{situationsReuse}}",
                        "settings": {
                            "productKey": "{ProductForEdit}"
                        },
                        "leadingSectionIdOrPath": "stateTest",
                    }
                }

Reuse components grouped into a subsection in a existing section can be moved within the section, using the SAP Visual Editor. All reuse components grouped together as a single section can also be moved within the section.

Hiding Reuse Components via API

You can hide reuse components using the extension API SetSectionHidden. Use this API only to hide reuse components, not to hide other sections.

sap.suite.ui.generic.template.ObjectPage.extensionAPI.ExtensionAPI.SetSectionHidden accepts only one boolean argument, either "true" or "false".

This API needs to be called in the component.js of the reuseable components.

Sample Code 
function fnRegisterOnPageDataLoaded(oExtensionAPI){
        oExtensionAPI.attachPageDataLoaded(function(oEvent){
            var oContextData = oEvent.context.getObject();
            (oExtensionAPI.setSectionHidden || jQuery.noop)(oContextData.ProductCategory !== "Notebooks"); 
        });
    }
